### Задание 1
Преимущества масштабирования

Активный master + пассивный slave:
Повышает отказоустойчивость (slave — резерв для master).
Разгружает master (чтение можно перенаправлять на slave).
Обеспечивает актуальную копию для восстановления и аналитики.

Master + несколько slave:
Горизонтальное масштабирование чтения (больше slave — больше одновременных запросов).
Балансировка нагрузки и высокая доступность.
Возможность распределять серверы по регионам для ускорения доступа.
